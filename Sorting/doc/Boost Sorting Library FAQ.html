Aren't the STL sorting functions good enough?

In a few cases. However, these functions are quicker and can do more, like sort within sort, and reverse sort.

What is a radix?

A radix is the base of the number system, for example, binary has radix 2.

Why not use a sortable interface class?

The reason for that is because it is easier for the users to define their own operators, it is faster, it takes up less memory, and it results in smaller code.
