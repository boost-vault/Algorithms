Introduction

Quicksort is a high-speed sorting algorithm that uses comparisons to sort objects. It is easier to implement than radix sort, and will work on objects of any value. It is a very good general prupose sorting algorithm. This implementation of quicksort is designed to avoid the worst case of O(N2 / 4) because it takes the median of the leftmost value, the rightmost value, and the middle value and uses the median as the partitioning element, so if the data is nearly sorted or reverse-sorted, it will not have as bad of a worst case.

Functions 
template<class T> void quicksort(T* a, int size)
Requires: that for any type T, T must be an integral type or T must inherit from boost::sortable (how to implement boost::sortable for any type)
Effects: that a is sorted from highest to lowest up to size
Complexity: best case: O(N log N), worst case: N2 / 4, where N is the number of elements
Danger: this has bad performance on sorted data
